Grid

Factory function for creating grids. It accepts optional hex settings that are passed directly to HexFactory. Several "shape" methods are exposed that return an array of hexes in a certain shape.

A grid is viewless, i.e.: it's an abstract grid with undefined dimensions. If you want to render a tangible grid, use a View factory (e.g. the DOM view).

Grid(hexSettings: Object?): Object
Parameters
hexSettings (Object?) Optional settings that apply to all hexes in the grid. See Hexfactory for possible properties.
Returns
Object: An object with methods.
Example
import { Grid, HEX_ORIENTATIONS } from 'Honeycomb'

const grid = Grid({
    size: 50,
    orientation: HEX_ORIENTATIONS.POINTY,
    template: hex => `<div class="hex">
        x: ${hex.x}
        y: ${hex.y}
        z: ${hex.z}
    </div>`
})

Hex

Factory function for creating hexes.

Coordinates not passed to the factory are inferred using the other coordinates:

  • When two coordinates are passed, the third coordinate is set to the result of Hex.thirdCoordinate(firstCoordinate, secondCoordinate).
  • When one coordinate is passed, the second coordinate is set to the first and the third coordinate is set to the result of Hex.thirdCoordinate(firstCoordinate, secondCoordinate).
  • When nothing or a falsy value is passed, all coordinates are set to 0.
Hex(coordinates: (Number | Object)?, y: Number?, z: Number?): Hex
Parameters
coordinates ((Number | Object)? = 0) The x coordinate or an object containing any of the x, y and z coordinates.
Name Description
coordinates.x Number? (default 0) The x coordinate.
coordinates.y Number? (default 0) The y coordinate.
coordinates.z Number? (default 0) The z coordinate.
y (Number? = 0) The y coordinate.
z (Number? = 0) The z coordinate.
Returns
Hex: A hex object. It always contains all three coordinates ( x , y and z ).
Example
Hex()            // returns hex( x: 0, y: 0, z: 0 )
Hex(1)           // returns hex( x: 1, y: 1, z: -2 )
Hex(1, 2)        // returns hex( x: 1, y: 2, z: -3 )
Hex(1, 2, -3)    // returns hex( x: 1, y: 2, z: -3 )
Hex(1, 2, 5)     // coordinates don't sum up to 0; throws an error

Hex({ x: 3 })    // returns hex( x: 3, y: 3, z: -3 )
Hex({ y: 3 })    // returns hex( x: 3, y: 3, z: -6 )
Hex({ z: 3 })    // returns hex( x: 3, y: -6, z: 3 )

HexFactory

Factory that produces a Hex function to create hexes with. It accepts optional hex settings that are used to create a "family" of hexes that can be used in a grid (or individually). This "family" of hexes all share the same prototype.

HexFactory($0: Object): Hex
Parameters
$0 (Object = {})
Name Description
$0.orientation (FLAT | POINTY)? (default FLAT) All hexes are either POINTY â¬¢ or FLAT â¬£.
$0.size Number? (default 1) Size of all hexes.
$0.origin Point? (default Point()) Used to convert the hex position to a point. Defaults to the top left.
$0.template Function? Template function that should return a (visual) representation of the hex. It gets passed the current hex when called. Could be an HTML string (e.g. '<div class="hex"></div>' ) that can be parsed by a Views.DOM instance. A View uses the hex's Hex#view method to call the template function and produce a view.
Returns
Hex: A function to produce hexes, all with the same prototype .
